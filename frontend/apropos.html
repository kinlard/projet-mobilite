Concept du projet

Eco-Escapade est une application de webmapping universitaire conçue pour évaluer l'accessibilité écologique des gares SNCF en France. L'objectif est de promouvoir la mobilité durable en mettant en lumière les gares les mieux connectées aux transports doux (vélo, véhicules électriques, covoiturage).

Public visé : Éco-voyageurs, étudiants et citoyens soucieux de leur empreinte carbone souhaitant planifier des déplacements en train de manière responsable.

Fonctionnalités clés :

  • Carte interactive avec visualisation des gares TGV et TER
  • Score écologique calculé automatiquement pour chaque gare
  • Zone piétonne de 10 minutes pour identifier les services accessibles à pied
  • Comparateur de gares avec suggestions d'alternatives mieux notées
  • Carnet de favoris avec sauvegarde locale
  • Données météo et qualité de l'air en temps réel

Méthodologie

Le cœur du projet repose sur un algorithme qui attribue un score écologique de 0 à 10 à chaque gare. Ce score est calculé dynamiquement en fonction des services de mobilité douce à proximité.

Formule de calcul :

Note = Bonus TGV + Points Vélos + Points Bornes IRVE + Points Covoiturage

Bonus TGV : +1 point si la gare est desservie par des trains TGV.

Parkings Vélos (rayon 800m, ~10 min à pied) :
→ +1 point par parking vélo détecté, plafonné à 7 points max.

Bornes IRVE (rayon 3 km) :
→ +0.5 point par borne de recharge électrique, plafonné à 2 points max.

Aires de Covoiturage (rayon 3 km) :
→ +0.5 point par aire de covoiturage, plafonné à 1 point max.

Échelle de notation :

0 - 3.9 : 🔴 Potentiel limité
4 - 6.9 : 🟠 Bon potentiel
7 - 10 : 🟢 Excellent

Note : La formule Haversine est utilisée pour calculer les distances géodésiques entre la gare et les équipements environnants.

Sources de données

  • API SNCF Open Data : Liste des gares de voyageurs avec coordonnées GPS, type (TGV/TER), et tracé des lignes ferroviaires (WFS GeoJSON). Données de propreté en gare et localisation des défibrillateurs.
  • OpenDataSoft (données OSM) : Parkings vélos en France (osm-france-bicycle-parking), bornes de recharge électrique IRVE (osm-france-charging-station), et aires de covoiturage (aires-covoiturage).
  • Open-Meteo API : Données météorologiques en temps réel (température, conditions) pour chaque gare.
  • OpenAQ API : Indice de qualité de l'air basé sur les stations de mesure proches (PM2.5, PM10, O3, NO2).
  • iNaturalist API : Données de biodiversité locale (espèces observées dans un rayon de 5 km autour des gares).
  • Wikipedia API : Récupération dynamique des photos des villes pour illustrer les gares.
  • Google Satellite Tiles : Fond de carte hybride satellite avec labels.

Architecture

Le backend Node.js/Express agit comme proxy API avec mise en cache (TTL 1h pour les données, 24h pour propreté/défibrillateurs). Il agrège les données de multiples APIs externes et les expose au frontend via des endpoints REST. Le frontend est une Single Page Application (SPA) utilisant Leaflet pour le rendu cartographique avec clusters optimisés et animations fluides (GPU compositing).
